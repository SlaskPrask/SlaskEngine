5. Designing the first scene

Our first scene

Scenes are essentially game screens or worlds. This scene will be our first world where the game starts in. Normally the first scene is a loading screen scene or the main menu scene.


world1.h


#ifndef WORLD1_H

#define WORLD1_H


#include <Slask.h>

using namespace slask;


#include "player.h"


class World1:public Scene

{

   SLASKSCENE

   

   private:
   Camera cam;

   Player *player;
   
   public:

   World1();

   ~World1();

};


#endif

Functions and methods provided by the engine are in bold text.


class World1:public Scene

{

   SLASKSCENE

All our custom scene classes should inherit the public Scene class and for the engine to handle your scenes you need to add the SLASKSCENE definition at the beginning of the class.


   private:
   Camera cam;

   Player *player;

Our scene will have a camera that will follow our player object. A camera is essentially a small region of the scene shown at once. We want to keep track of our player object within this scene so we add it as a pointer to it.


   public:

   World1();

   ~World1();

};

The constructor will contain setting up this scene and all the objects it contains. For more complex game scenes it can contain methods to load scene data from external files. The destructor will contain unloading the scene and it's resources.
