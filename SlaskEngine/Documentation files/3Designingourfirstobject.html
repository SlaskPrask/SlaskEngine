Designing our first object

Objects are everything showing up, moving and interacting in the game. Our player will be it's own object class. If you were to make an enemy, every enemy type will have it's own class in the game there can be multiple enemies of the same type and class.


world1.h


#ifndef PLAYER_H

#define PLAYER_H


#include <Slask.h>

using namespace slask;


class Player:public Object

{

   SLASKOBJECT

   

   private:

   Sprite *sprite;

   
   public:

   Player();

   

   void run();

   void draw();

   

   ~Player();

};


#endif

Functions and methods provided by the engine are in bold text.


class Player:public Object

{

   SLASKOBJECT

For the engine to handle your objects give them the public parent Object and add the SLASKOBJECT definition at the beginning of the class.


   Player();

   ~Player();

};

The constructor and destructor will contain the object's code on creation and deletion. The constructor will essentially set up the object's variables.
